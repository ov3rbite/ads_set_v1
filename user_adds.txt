Assets You Need to Provide
==========================

Each product landing page has placeholder content that works out of the box,
but you'll want to swap in real brand assets before running ads.

BUKSA (static/buksa/images/)
-----------------------------
- hero-screenshot.png    — Editor UI mockup or writing interface screenshot
- og-image.png           — 1200x630 Open Graph image for social sharing

HOTGOSS (static/hotgoss/images/)
---------------------------------
- app-store-badge.svg    — Apple App Store download badge
- play-store-badge.svg   — Google Play download badge
- og-image.png           — 1200x630 Open Graph image

SAMESKY (static/samesky/images/)
---------------------------------
- og-image.png           — 1200x630 Open Graph image

TASKMATIC (static/taskmatic/images/)
-------------------------------------
- taskmatic-wordmark-white.svg  — Already available in ads/html/taskmatic-ads/
- taskmatic-wordmark-black.svg  — Already available in ads/html/taskmatic-ads/
- logo.png                      — Already available in ads/html/taskmatic-ads/
- og-image.png                  — 1200x630 Open Graph image

ENVIRONMENT VARIABLES (set in Render or .env)
----------------------------------------------
- DJANGO_SECRET_KEY   — Auto-generated by Render, or set manually
- DASHBOARD_KEY       — Secret key for /_dashboard/ access
- ALLOWED_HOSTS       — Comma-separated hostnames
- DEBUG               — "False" in production
- DATABASE_URL        — Optional: Postgres connection string (defaults to SQLite)
